## closures and concurrency

闭包捕获的是变量的引用，for 循环很快就结束了，在goroutine中，可能三次都打印的是c，也可能是其他的；

与我们预期稳定输出为 a，b，c 相背

```go
func main() {
    done := make(chan bool)

    values := []string{"a", "b", "c"}
    for _, v := range values {
        go func() {
            fmt.Println(v)
            done <- true
        }()
    }

    // wait for all goroutines to complete before exiting
    for _ = range values {
        <-done
    }
}

// output：
/*
c
c
c
*/
```

正确的做法：

- 在启动时将当前值绑定到每个闭包上，在每次循环时新增一个变量进行传递

```go
for _, v := range values {
    go func(u string) {
        fmt.Println(u)
        done <- true
    }(v)
}
```
